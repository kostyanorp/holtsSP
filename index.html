Git - Система контролю версій

Що таке система контролю версій?

Система контролю версій (VCS) — це інструмент, що дозволяє зберігати історію змін у файлах, відслідковувати, хто і коли вніс правки, а також повертатися до попередніх версій. Приклади: Git, SVN, Mercurial.

Що таке Git?

Git — це розподілена система контролю версій, створена Лінусом Торвальдсом у 2005 році для розробки ядра Linux. Вона дозволяє командній роботі над проєктами, зручне злиття змін, гілкування та відновлення історії.

Основні можливості та переваги Git

  • Розподілене зберігання (копія історії є у кожного користувача).
  • Швидке перемикання між гілками (branching).
  • Можливість працювати офлайн.
  • Гнучке злиття змін (merge, rebase).
  • Висока швидкість роботи навіть у великих проєктах.
  • Безпека завдяки хешуванню SHA-1.

10 базових команд Git

  • git init – створює новий репозиторій.
  • git clone [url] – клонування існуючого репозиторію.
  • git status – перевірка стану файлів.
  • git add [файл] – додавання файлу до індексу.
  • git commit -m "повідомлення" – фіксація змін у репозиторії.
  • git log – перегляд історії комітів.
  • git branch – перегляд або створення нових гілок.
  • git checkout [гілка] – перемикання на іншу гілку.
  • git merge [гілка] – злиття гілки з поточною.
  • git push / git pull – відправка та отримання змін з віддаленого репозиторію.
